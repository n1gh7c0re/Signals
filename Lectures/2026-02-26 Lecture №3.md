# Конспект лекции №3
## **Обработка и интерпретация сигналов**  
**Тема: Синдромное декодирование линейных кодов. Смежные классы. Границы параметров кодов (Хэмминга и Варшамова–Гилберта)**

## 1. Постановка задачи декодирования

При передаче данных по двоичному симметричному каналу (ДСК) кодовое слово $c \in C$ может быть искажено вектором ошибки $e$. На приемной стороне мы получаем вектор:


$$y = c \oplus e$$

**Цель декодирования по максимуму правдоподобия (MLD):** найти такое кодовое слово $\hat{c}$, которое минимизирует расстояние Хэмминга $d(y, \hat{c})$. В двоичном случае это эквивалентно поиску вектора ошибки $\hat{e}$ с минимальным весом Хемминга $\omega(e)$, такого что $y = \hat{c} \oplus \hat{e}$.

---
## 2. Смежные классы (Косеты) и Стандартная таблица

Для эффективного поиска ошибки всё пространство $F_2^n$ (объемом $2^n$) разбивается на непересекающиеся подмножества относительно кода $C$.

**Определение.** Смежным классом (косетом) кода $C$ по вектору $a \in F_2^n$ называется множество:


$$a \oplus C = \{ a \oplus c \mid c \in C \}$$

### Свойства смежных классов:

1. **Разбиение:** Пространство $F_2^n$ разбивается на $2^{n-k}$ непересекающихся смежных классов.
2. **Мощность:** Каждый класс содержит ровно $|C| = 2^k$ векторов.
3. **Лидер класса:** Вектор с минимальным весом Хемминга в смежном классе. Он представляет собой наиболее вероятную ошибку, которую может исправить данный код в этом классе.

### Стандартная таблица декодирования (Standard Array)

Это способ визуализации всех возможных векторов в пространстве, сгруппированных по ошибкам.

| Лидеры ($e$) | $c_0 = 0...0$ | $c_1$ | ... | $c_{2^k-1}$ |
| --- | --- | --- | --- | --- |
| **$e_0 = 0$** | $c_0$ (весь код $C$) | $c_1$ | ... | $c_{2^k-1}$ |
| **$e_1$** | $e_1 \oplus c_0$ | $e_1 \oplus c_1$ | ... | $e_1 \oplus c_{2^k-1}$ |
| ... | ... | ... | ... | ... |
| **$e_{2^{n-k}-1}$** | $e_{last} \oplus c_0$ | ... | ... | $e_{last} \oplus c_{2^k-1}$ |

## 3. Синдромное декодирование

Хранение стандартной таблицы требует $2^n$ ячеек памяти, что непрактично. Синдромное декодирование позволяет сжать таблицу до $2^{n-k}$ строк.

**Определение.** Синдромом принятого вектора $y$ называется вектор $S$ длины $r = n - k$:


$$S = y \cdot H^T$$

### Фундаментальные свойства синдрома:

1. **Зависимость только от ошибки:** $S = (c \oplus e)H^T = cH^T \oplus eH^T = \mathbf{0} \oplus eH^T$. Синдром не зависит от того, какое именно кодовое слово передавалось.
2. **Инвариантность в классе:** Все векторы одного смежного класса имеют один и тот же синдром.
3. **Биекция:** Существует взаимно однозначное соответствие между синдромами и лидерами смежных классов.


### **Практический алгоритм (Offline + Online)**

**Offline (один раз):**
- Для всех 2ⁿ возможных векторов **e** вычислить **S = e · Hᵀ**.
- Сгруппировать по синдромам.
- Для каждого синдрома выбрать **лидер** — вектор **минимального веса**.

**Online (при приёме y):**
- Вычислить **S = y · Hᵀ**.
- По таблице найти лидер **e**.
- Исправить: **ĉ = y ⊕ e**.

**Преимущество:** вместо $2^{n}$ элементов храним только $2^{n−k}$ записей.

## 4. Геометрическая интерпретация и сферы Хэмминга

Если представить кодовые слова как центры в n-мерном пространстве, то декодирование — это процесс определения того, в «сферу влияния» какого центра попал принятый вектор.

* **Сфера Хэмминга** радиуса $t$ вокруг слова $c$: множество всех векторов $y$, таких что $d(c, y) \le t$.
* **Объем сферы $V(n, t)$:** количество векторов в сфере радиуса $t$:

$$V(n, t) = \sum_{i=0}^{t} \binom{n}{i}$$

**Радиус покрытия** кода — максимальный вес вектора, который ещё декодируется в данное кодовое слово.

**Идеал** — **плотная сферическая упаковка**: сферы не пересекаются и полностью заполняют пространство.

## 5. Границы параметров кодов

### 5.1. Граница Хэмминга (Верхняя граница / Плотная упаковка)

Ни один код не может иметь больше кодовых слов $M$, чем позволяет суммарный объем непересекающихся сфер вокруг них:

$$M \cdot \sum_{i=0}^{t} \binom{n}{i} \le 2^n$$

**Совершенные коды** — коды, для которых достигается **равенство** (сферы полностью заполняют пространство).

**Примеры совершенных кодов:**
- Все коды Хэмминга (t = 1).
- Код Голея (23,12,7) — исправляет t = 3 ошибки.

**Проверка кода Голея:**

$$2^{12} \cdot (\binom{23}{0} + \binom{23}{1} + \binom{23}{2} + \binom{23}{3}) = 2^{12} \cdot 2048 = 2^{23}$$

Равенство выполняется $\implies$ код Голея совершенен.

### 5.2. Граница Варшамова–Гилберта (Нижняя граница / Конструктивная)

Эта граница гарантирует, что «хорошие» коды существуют. Линейный $(n, k)$-код с минимальным расстоянием $d$ существует, если:


$$2^{n-k} > \sum_{i=0}^{d-2} \binom{n-1}{i}$$

**Алгоритм построения:**

Мы строим проверочную матрицу $H$ столбец за столбцом. Новый $i$-й столбец можно добавить в $H$ только если он **не является** линейной комбинацией любых $(d-2)$ уже имеющихся столбцов. Если количество «запрещенных» комбинаций меньше, чем общее число доступных ненулевых векторов $2^{n-k}-1$, мы всегда сможем найти новый столбец.

---

## 6. Резюме: Сравнение границ

* **Граница Хэмминга:** говорит, что кодов лучше, чем «это», быть **не может** (необходимое условие).
* **Граница Варшамова–Гилберта:** говорит, что код с такими параметрами **точно есть** (достаточное условие).
